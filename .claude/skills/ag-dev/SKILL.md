# ag-dev 스킬

`/ag-dev <작업내용>` 형식으로 호출합니다.

## 역할

`ag-dev` 뒤에 오는 명령은 **에이전트를 활용한 개발 작업** 요청입니다.

다음 에이전트를 적절히 활용하여 작업을 수행하세요:

- **qa-tester**: 버그 발견, 엣지케이스, UX 문제, 회귀 검증이 필요할 때 사용
- **senior-architect-refactorer**: 아키텍처 설계, 코드 리뷰, 리팩토링, 구조 개선 — **분석 후 직접 코드 구현까지 수행**
- **senior-product-planner**: 기능 정의, 요구사항 분석, 구현 범위 결정이 필요할 때 사용

작업 성격에 따라 하나 또는 여러 에이전트를 순서대로 활용하세요.

> **버그 수정 시 권장 순서**: qa-tester(원인 분석) → senior-architect-refactorer(수정 방향 설계 + 코드 직접 구현)

## 한국어 주석 규칙 (필수)

코드를 작성하거나 수정할 때 **반드시** 다음 규칙을 따르세요:

1. **모든 함수/메서드**에 한국어로 목적과 동작을 설명하는 주석을 달아야 합니다.
2. **복잡한 로직**이나 비즈니스 규칙이 있는 코드 블록에는 한국어 인라인 주석을 추가합니다.
3. **변수명이 직관적이지 않은 경우** 한국어로 설명을 추가합니다.
4. 주석은 **코드를 처음 보는 사람도 이해할 수 있을 정도로** 충분히 명확하게 작성합니다.
5. 단순히 코드를 반복하는 주석은 지양하고, **왜(why)** 이렇게 작성했는지를 설명합니다.

### 주석 예시

```typescript
// 사용자 인증 토큰을 검증하고 만료 여부를 확인하는 함수
// 토큰이 유효하지 않으면 null을 반환하여 인증 실패를 알림
function validateToken(token: string): User | null {
  const decoded = jwt.verify(token, SECRET_KEY);

  // 토큰 만료 시간이 현재 시간보다 이전이면 무효 처리
  if (decoded.exp < Date.now() / 1000) {
    return null;
  }

  return decoded.user;
}
```

## 노션 기록 규칙 (필수)

Notion 자동 기록은 **Agent 도구가 실제로 호출될 때만** PostToolUse 훅이 실행됩니다.

### 반드시 지켜야 할 규칙

1. **모든 /ag-dev 작업은 반드시 에이전트를 통해 분석·리뷰 단계를 거쳐야 합니다.**
   - 코드를 직접 읽고 수정하는 것만으로는 Notion에 기록되지 않습니다.
   - 아무리 간단한 버그 수정이라도 qa-tester 또는 senior-architect-refactorer를 먼저 호출하세요.

2. 에이전트 프롬프트 작성 시 다음을 포함하세요:
   - 에이전트가 수행할 **태스크를 명확하게** 기술 (Notion 태스크 컬럼에 기록됨)
   - 에이전트 결과의 **핵심 결론과 결정 사항**을 명시적으로 출력하도록 지시 (Notion 결과 요약 컬럼에 기록됨)

> Notion 데이터베이스: `Agent 협업 로그` (프로젝트 페이지 내 자동 생성)

## 작업 흐름

1. 사용자의 요청을 분석하여 어떤 에이전트가 필요한지 판단합니다.
2. **에이전트를 반드시 호출**하여 분석/설계/리뷰를 수행합니다 → **Notion에 자동 기록**.
   - 에이전트를 생략하고 직접 코드를 수정하면 Notion에 기록되지 않습니다.
3. 실제 코드 작성 시 위의 한국어 주석 규칙을 반드시 적용합니다.
4. 결과물을 사용자에게 명확하게 전달합니다.
